# 中文共指消岐方案

本目录给出一个面向中文实体链接场景的共指消岐基线实现，并附带可运行测试。

## 设计结论

规划文档里的主线是 `HanLP NER -> 候选生成 -> BGE 消歧 -> NIL -> Coreferee 共指消解`。这个结构本身是合理的，但其中的共指模块不适合直接按英文方案落到中文：

1. `Coreferee` 并不是一个可靠的中文共指方案，中文模型名 `zh_core_web_sm` 也不是一个可直接假定可用的稳定依赖。
2. `FastCoref + XLM-RoBERTa` 这类组合更容易遇到分词、span 对齐、候选边界和中文代词集合的问题，尤其在“先做实体链接，再做共指”的治理场景里，维护成本高于收益。
3. 当前课题更强调可追溯、可审计、可回放，因此共指模块应该优先可解释、可控、可替换，而不是先追求大模型式端到端。

## 推荐方案

采用“中文规则共指 + 可插拔模型重排”的两层结构：

1. 第一层用确定性规则解决高频、低风险、可审计场景。
2. 第二层预留模型接口，只在规则置信度不足时触发。
3. 输出必须保留 antecedent、命中规则、置信度和失败原因。

## 针对质疑的回应

下面这些质疑里，大多数都成立，但它们批评的是“只靠规则去做完整共指”的方案；如果把目标收窄为“以代词回链为主的中文共指基线”，结论就会不同。

1. 中文代词天然歧义，单靠规则无法区分多实体场景。这个质疑是合理的。中文代词本来就需要上下文消歧，尤其“它”同时可指机构、设备、抽象对象。解决办法不是坚持纯规则，而是把规则只用于召回和强约束：先用实体类型、最近激活实体、别名表和句内位置缩小候选，再用可插拔模型或 reranker 在候选里排序。也就是说，规则负责“找得到候选”，模型负责“候选之间怎么选”。

2. 零主语省略在中文里会让规则失效。这个质疑也合理，但它不意味着方案不可做，而是意味着不能把“全句无先行词”的情况都交给简单正则。处理方式是把共指模块放在实体链接流水线里，利用前文已链接实体建立激活栈，再结合句法线索、篇章窗口和业务模板判断隐式主语；如果前文确实没有任何可用先行实体，就应当显式返回 NIL，而不是强行猜测。

3. 长距离跨句指代会让近距离规则失效。这个质疑合理。纯最近优先会在多段落文本里出错。解决方式是给候选打分时同时考虑句距、段距、实体激活时间和话题保持，而不是只看“最近”。工程上可以保留一个滑动窗口加一个长程记忆池：短窗口负责高精度，长程池负责召回，最终由打分器统一排序。

4. 无法区分实体类型，容易错匹配非业务实体。这个质疑合理，而且是中文共指里最常见的错误之一。解决方式是把实体类型作为硬约束或强特征：人称代词优先找 PERSON，组织/机构代词优先找 ORG，设备/产品代词优先找非人实体；如果类型不一致，直接降权或排除。这样可显著减少“小李”和“国家电网项目”这类误绑。

5. 无法处理简称、别名、同义替换。这个质疑合理，但它不是纯共指问题，而是共指与知识库对齐的交叉问题。解决办法是让别名表先做标准化映射，再把标准实体放入激活栈；前文“国家电网”和后文“国网”不应靠代词规则硬猜，而应由知识库别名表把它们统一到同一个实体 ID，再由共指模块回链到这个已知实体。

6. 复杂并列、从句句式规则难以覆盖。这个质疑合理，说明不能指望靠大量 if-else 覆盖所有句式。解决方式是把规则范围限定在高频、低风险、收益最高的代词回链场景；对于复杂并列、嵌套从句、歧义很高的文本，交给句法分析器或模型 reranker。工程上不要追求“所有句式都写规则”，而要追求“规则覆盖 80% 高频场景，其余交给可回退链路”。

7. 边界模糊代词无统一判定标准。这个质疑合理。像“该企业、该公司、前者、后者、本方”这类准代词，本质上已经接近篇章指代或话语指代。解决方案是拆分词表层级：高置信代词、名词性指代、篇章指代三层处理；第一层用规则直接回链，第二层用类型和别名增强，第三层只在有足够上下文时启用模型，否则宁可返回 NIL，也不要硬判。

## 这套方案真正解决什么

我们主要不是做“完整开放域共指”，而是解决实体链接链路里的代词问题：

1. 在实体已经被识别、部分已链接的前提下，把代词和名词性指代回链到正确实体。
2. 对能确定的结果给出可追溯依据，对不能确定的结果明确返回 NIL。
3. 在中文业务文本里优先服务“机构、项目、人物、产品”的代词消歧，而不是泛化到开放域所有共指。

因此，外部质疑对“纯规则共指”是成立的，但对我们现在这个“规则基线 + 别名表 + 类型约束 + 可插拔中文兜底”的方案，只能算是提示边界，不是推翻方案本身。

这里的“可插拔中文兜底”不是指 Coreferee，而是指可以替换为中文共指重排器、跨句判别器、句法特征打分器或低置信度 LLM 兜底。

### 规则层覆盖范围

- 人称代词：他、她、他们、她们、它、它们。
- 组织代词/名词性指代：该公司、该集团、该校、该院、该局、本公司、本集团、这家公司、这所学校。
- 基于最近优先、类型一致、别名匹配、同句优先的消歧。

### 适配实体链接流水线的方式

共指模块不直接替代 NER 或实体链接，而是只做“已识别 mention”的回链：

1. 已链接的实体 mention 进入激活队列。
2. 代词/名词性指代只在激活队列里找候选。
3. 找不到候选时返回 NIL，并记录原因。
