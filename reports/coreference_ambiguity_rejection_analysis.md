# 最近组默认 + 歧义拒绝器离线分析

## 1. 实验边界

本报告只基于 `coreference_challenge_dev_v2.json` 的 39 条开发样例，不运行 blind holdout，不修改正式共指决策、API 或 gold。拒绝器仅在离线脚本中生效：默认保留正式规则的最近合法协调组，只有命中强歧义组合条件时才输出 NIL。

## 2. 新增配对样例

新增 `ambiguity_rejection_pilot` 共 8 条：4 条最近组显式主体持续正例，4 条新显式主体/事件切换 NIL。每个负例均保留两组协调实体、连接词和集合代词，仅改变最近组后是否继续作为显式主体、是否出现新主体与事件重置。

## 3. 可解释特征

|特征|离线计算方式|作用|
|-|-|-|
|`explicit_subject_continuation`|最近组的全部成员表面词在组结束与代词之间再次出现|正向保留证据。|
|`explicit_subject_switch`|组后到代词前出现新的已链接命名实体，且邻近通用动作词如“随后”“宣布”“发布”“负责”等|拒绝风险证据。|
|`event_chain_continuity`|按通用事件类别词典识别“签署/发布/实施”“建设/验收/运营”等至少两类连续动作|正向辅助证据，不单独决策。|
|`event_reset_signal`|识别“与此同时”“另一方面”“项目转向”等通用重置信号|拒绝风险证据。|
|`candidate_score_gap`|最近候选与次近候选的正向证据数差值|阈值扫描输入。|
|`evidence_strength`|数量匹配、类型匹配、主体持续、事件连续的累计证据数|避免只凭多候选直接拒绝。|

强歧义拒绝要求：存在多个合法协调组、最近组没有显式主体持续、存在新主体切换或事件重置、候选差值不高于阈值，且最近组证据强度不高于最小证据上限。它不重新排列候选组。

## 4. 最佳阈值与扫描边界

保守选择为：

```text
candidate_score_gap = 0.0
minimum_evidence_count = 3
```

该组合在完整 v2 上为 26/39。多个更宽松的 gap 值在当前小样本上产生同分，因此 gap 参数尚不稳定；保留 `0.0` 作为最保守的并列最优值。完整 25 组扫描记录在 `coreference_discourse_experiment.json` 的 `ambiguity_threshold_scan`。

## 5. Baseline 与拒绝器对比

|范围|方案|总体|正例保留|NIL 正确|False NIL|False Positive|
|-|-|-:|-:|-:|-:|-:|
|完整 v2（39）|Baseline|22/39|16/22|6/17|4|11|
|完整 v2（39）|拒绝器|26/39|16/22|10/17|4|7|
|新增配对子集（8）|Baseline|4/8|4/4|0/4|0|4|
|新增配对子集（8）|拒绝器|8/8|4/4|4/4|0|0|

拒绝器相对完整 v2 baseline 多正确 4 条，未新增 false NIL，NIL 正确数增加 4 条。新增配对样例中，主体持续的 4 条正例均保留；新主体切换的 4 条 NIL 均被拒绝。

## 6. 风险与结论

结论为 **PROMISING**，但仅表示这条“最近组默认 + 强歧义拒绝”路径值得继续扩充 Challenge Dev v2。它尚未通过独立 blind holdout，且现有同句歧义 NIL（未带显式切换信号）仍可能被最近组策略误选。

不应据此接入正式 Pipeline。下一步应扩大不依赖当前词面的主体持续/切换对照，并检验事件类别词典在不同领域中的误报率。

## 7. 跨领域鲁棒性补充

第二轮新增 16 条 `robustness_domain_pilot`，将 v2 扩展至 55 条。离线拒绝器在新增子集为 16 / 16，完整 v2 为 **42 / 55**；baseline 为 **32 / 55**。正例保持 24 / 30，NIL 正确数由 8 / 25 增至 18 / 25。完整领域与误报明细见 `reports/coreference_ambiguity_rejection_domain_analysis.md`。

该提升满足开发集的继续门槛，但新增样本仍是围绕同一可解释假设构造的开发数据；在未运行独立 blind holdout 前，结论仍限定为 **PROMISING（离线实验）**，不构成正式 Pipeline 接入结论。
