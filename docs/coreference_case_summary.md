## 1. 本阶段完成内容

本阶段完成 `entity_linker/coreference.py` 规则共指模块，实现中文代词和名词性指代的轻量解析能力，并在主 Pipeline 中封装 `enable_coreference` 调用逻辑。

模块边界保持为增量实现：

- 不替代 NER、Candidate Generation、Disambiguation；
- 只处理已识别 mention 中的代词或指代短语；
- 先行词只来自已经链接到标准实体的 mention；
- 集合指代、无先行词、低置信场景返回 NIL；
- 输出保留 `entity_id`、`entity_name`、`confidence`、`evidence`、`is_nil`，便于 trace 与人工复核。

## 5. 错误案例统计

当前评测集未出现错误案例：

- wrong = 0
- 主要原因是本阶段评测集聚焦规则可覆盖的高置信场景；
- 后续仍需扩大样本覆盖，以验证更复杂的跨句、多实体、长距离回指。

## 6. 后续优化方向

下一阶段建议重点优化：

1. 增加更复杂的多实体竞争样本，例如同一段落内连续出现 3 个以上 ORG 后再出现 `该公司`。
2. 增加跨段落、长距离回指样本，测试 `max_sentence_gap` 对召回率的影响。
3. 增加类型细分能力，例如 `该校` 更偏向教育机构，`该院` 可能指医院或研究院。
4. 对集合指代补充结构化输出能力，例如允许返回多个 `entity_id`，而不是统一 NIL。
5. 后续可接入 LLM 或轻量模型作为规则不确定场景的重排器，但默认仍应保持关闭。
